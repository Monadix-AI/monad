# Dev Shell Binding Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `bun install` initialize the current worktree and expose its source CLI as `monad` in direnv-authorized shells.

**Architecture:** The root `postinstall` script invokes the existing idempotent dev initializer. A focused dev-init helper renders and installs a project-local CLI shim in `.dev/bin`, while `.envrc` loads `.env.local` and prepends that bin directory after selecting the pinned Bun version. Repository trust remains explicit: neither install nor dev-init invokes `direnv allow`.

**Tech Stack:** Bun 1.3.14, TypeScript, Bun test, direnv shell configuration.

## Global Constraints

- Preserve the current mise-based Bun version selection in `.envrc`.
- Never invoke `direnv allow` automatically.
- Keep dev initialization idempotent and scoped to the current checkout/worktree.
- Preserve unrelated dirty TUI changes in the worktree.
